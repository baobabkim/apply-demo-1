## 1. 프로젝트 배경 (Background)

현재 우리 C2C 플랫폼은 신규 유저 유입은 꾸준하나, **첫 방문 후 7일 이내에 다시 앱을 방문하는 유저의 비율(Retention)이 하락**하고 있음. 특히, 물건을 구경하는 유저(Viewer)는 많으나 실제로 판매자와 대화를 시작하는 유저(Chatter)로의 전환이 병목 현상을 보임.
 
* **현상:** 유저의 앱 체류 시간은 유지되나, 실질적인 인터랙션(채팅, 거래)으로 이어지지 않음.
* **가설:** 유저가 물건의 상태나 판매자의 신뢰도를 판단할 정보가 부족하여 '채팅하기' 버튼 클릭을 망설일 것이다.

## 2. 목적 및 목표 (Goals)

* **목적:** 유저 이탈 구간을 데이터로 특정하고, 전환율을 높일 수 있는 실험 가설을 도출함.
* **핵심 목표 (KPI):**
1. **D+7 리텐션 5%p 개선**을 위한 핵심 동인(Driver) 발견.
2. **상세 페이지 -> 채팅 시작 전환율(CVR) 10% 향상**을 위한 UI/UX 개선안 제안.

## 3. 타겟 유저 (Target Audience)

1. **신규 유저:** 앱 설치 후 첫 거래를 경험하지 못한 유저.
2. **잠재적 이탈 유저:** 최근 3일간 상품 상세 페이지를 5회 이상 조회했으나 채팅을 시작하지 않은 유저.

---

## 4. 데이터 분석 및 기능 요구 사항 (Analysis & Requirements)

### 4.1. 유저 행동 로그 분석 (Log Analysis)

* **요구 사항:** 유저가 앱 진입 후 이탈하기까지의 전체 Funnel을 시각화한다.
* **세부 분석 내용:**
* 메인 홈 -> 검색 -> 상세 페이지 -> 채팅 버튼 클릭 -> 실제 채팅 발송 단계별 이탈률 산출.
* 이탈 유저와 잔존 유저 간의 행동 패턴 차이(예: 검색어 유무, 사진 슬라이딩 횟수 등) 비교.

### 4.2. 리텐션 코호트 분석 (Cohort Analysis)

* **요구 사항:** 특정 행동(Action)이 리텐션에 미치는 영향을 분석하여 'Aha-moment'를 정의한다.
* **세부 분석 내용:**
* 첫 가입 후 24시간 이내에 '동네 인증'을 완료한 유저의 리텐션 비교.
* 첫 거래 성공 유저 vs 미성공 유저의 재방문 주기 분석.

### 4.3. 신규 기능 제안 (Feature Proposal)

* **제안 내용:** **'판매자 평균 응답 시간 및 신뢰 지표'** 노출 기능.
* **기능 상세:** 상세 페이지 상단에 "보통 10분 이내 응답하는 판매자입니다" 또는 "매너 점수가 높은 판매자입니다" 배지 노출.
* **기대 효과:** 정보 불확실성을 해소하여 유저가 부담 없이 채팅을 시작하도록 유도.

---

## 5. 실험 설계 (A/B Test Design)

도출된 기능 제안을 검증하기 위한 실험 설계입니다.

* **실험 대상:** 전체 유저의 20% (A그룹: 기존 페이지 / B그룹: 신뢰 지표 노출 페이지).
* **핵심 지표:** 상세 페이지 내 '채팅하기' 버튼 클릭률(CTR).
* **보조 지표:** 채팅 시작 후 실제 거래 완료율, CS 문의 건수.
* **기간:** 최소 2주 (통계적 유의성 확보를 위한 기간).

## 6. 기술 스택 및 도구 (Tech Stacks)

* **데이터 추출:** SQL (BigQuery 또는 PostgreSQL)
* **데이터 분석:** Python (Pandas, Scipy, Statsmodels)
* **시각화:** Tableau 또는 Preset (공고 기반)
* **협업 도구:** Slack, Confluence (문서화)

## 7. 기대 결과 (Expected Impact)

1. **데이터 기반 의사결정:** 감에 의존한 기능 업데이트가 아닌, 로그 분석을 통한 문제 지점 타격.
2. **유저 경험 최적화:** 유저가 신뢰할 수 있는 정보를 제공함으로써 플랫폼 내 거래 활성화.
3. **수익성 개선:** 리텐션 향상을 통한 마케팅 비용(CAC) 절감 효과.
